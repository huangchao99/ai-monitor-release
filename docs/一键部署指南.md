# AI Monitor 一键部署指南

本文档描述如何使用发布包配套的 `install-ai-monitor.sh`，在新部署机器上用一次命令完成安装、初始化、配置和基础自检。

## 1. 开发机打包

进入发布工作区：

```bash
cd /home/hzhy/ai-monitor-release
```

执行打包：

```bash
./scripts/package-release.sh 2026.04.10-rk3576-v2
```

打包完成后，产物默认位于：

```bash
dist/ai-monitor-release-2026.04.10-rk3576-v2/
```

同时会在 `dist/` 目录下生成一份一键部署脚本：

```bash
dist/install-ai-monitor.sh
```

将发布目录压缩：

```bash
cd /home/hzhy/ai-monitor-release/dist
tar -czf ai-monitor-release-2026.04.10-rk3576-v2.tar.gz ai-monitor-release-2026.04.10-rk3576-v2
```

## 2. 复制到部署机

把以下两个文件复制到部署机的同一目录：

```bash
ai-monitor-release-2026.04.10-rk3576-v2.tar.gz
install-ai-monitor.sh
```

例如复制到部署机的 `/root/ai-monitor-deploy/`。

## 3. 部署机执行一键安装

进入这两个文件所在目录：

```bash
cd /root/ai-monitor-deploy
```

执行：

```bash
sudo ./install-ai-monitor.sh ai-monitor-release-2026.04.10-rk3576-v2.tar.gz
```

如果当前目录下只有一个 `ai-monitor-release-*.tar.gz`，也可以省略压缩包参数：

```bash
sudo ./install-ai-monitor.sh
```

## 4. 脚本会自动完成什么

脚本会依次完成以下动作：

1. 解压发布包到临时目录
2. 执行包内 `scripts/install.sh`
3. 执行包内 `scripts/init-db.sh`
4. 将配置文件补齐到 `/etc/ai-monitor/`
5. 将 service 文件安装到 `/etc/systemd/system/`
6. 将 nginx 配置安装到 `/etc/nginx/sites-available/ai-monitor.conf`
7. 创建 `/etc/nginx/sites-enabled/ai-monitor.conf` 软链接
8. 执行 `nginx -t`
9. 重启 nginx
10. 检测 ZLMediaKit 是否已准备完成
11. 如果 ZLMediaKit 可用，则启动业务服务并执行 `health-check.sh`

## 5. 配置覆盖策略

`/etc/ai-monitor/` 采用保守模式：

- 如果目标文件不存在，则从发布包复制
- 如果目标文件已经存在，则保留现场配置，不覆盖

涉及文件包括：

- `/etc/ai-monitor/backend.env`
- `/etc/ai-monitor/python.env`
- `/etc/ai-monitor/infer.env`
- `/etc/ai-monitor/zlm.env`
- `/etc/ai-monitor/infer/server.json`

## 6. ZLMediaKit 启动策略

脚本会读取：

- `/etc/ai-monitor/zlm.env`

并检查以下两个条件：

- `AI_MONITOR_ZLM_BIN` 指向的可执行文件存在
- `AI_MONITOR_ZLM_CONFIG_PATH` 指向的配置文件存在

只有这两个条件都满足时，才会继续启动：

- `zlmediakit.service`
- `infer-server.service`
- `ai-monitor-python.service`
- `ai-monitor-backend.service`

否则会跳过这些服务，并输出提示信息。

## 7. 部署前提

部署机需要预先具备这些基础条件：

- 已安装 `systemd`
- 已安装 `nginx`
- 已安装 `sqlite3`
- 已安装 `curl`
- 已安装 `tar`
- 已安装 `rsync`
- 已安装 Python 3
- 已具备 RKNN Runtime / MPP / RGA 等基础环境
- 如需启动视频流链路，已正确安装并配置 ZLMediaKit

补充说明：

- `zlmediakit.service` 当前按 `hzhy` 用户运行
- 因此 `/opt/ZLMediaKit` 及其 `log/` 目录必须对 `hzhy` 可写
- 基础环境安装脚本会自动将 `/opt/ZLMediaKit` 修正为 `hzhy:hzhy`

## 8. 常见结果说明

如果脚本输出类似：

```text
[install-ai-monitor] ZLMediaKit not ready, skipped starting:
```

说明应用版本已经安装完成，但因为 ZLMediaKit 基础环境未就绪，所以未启动完整服务栈。此时需要先修正：

- `/etc/ai-monitor/zlm.env`
- `AI_MONITOR_ZLM_BIN`
- `AI_MONITOR_ZLM_CONFIG_PATH`

修正后可手动执行：

```bash
sudo systemctl restart zlmediakit.service infer-server.service ai-monitor-python.service ai-monitor-backend.service
sudo /opt/ai-monitor/current/scripts/health-check.sh
```

## 9. 推荐操作习惯

- 每次发版都使用明确版本号，例如 `2026.04.10-rk3576-v2`
- 压缩包和 `install-ai-monitor.sh` 始终放在同一目录
- 首次部署完成后，检查 `systemctl status`、`nginx -t` 和健康检查结果
- 如现场已有配置，不要手动删除 `/etc/ai-monitor/` 下的文件，否则下次会重新回填默认模板
