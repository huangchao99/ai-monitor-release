# HiFleet智能视频监控系统使用说明书

## 1. 这是什么软件

这是一套用于视频监控和智能识别的软件。它可以接入摄像头画面，对人员行为或安全事项进行自动识别，并在发现异常时生成告警、保存截图、播放语音、上传告警记录。

本说明书面向普通用户，重点说明“怎么用”。不要求您了解编程、算法或服务器知识。

系统界面左侧主要菜单包括：

- 控制台
- 设备管理
- 任务管理
- 事件告警
- 算法模型
- 模型测试
- 定位管理
- 语音报警
- 报警上传
- 系统管理

## 2. 开始使用前，请先确认

首次使用前，请确认以下条件已经具备：

- 软件已经安装完成，并能正常打开网页界面
- 摄像头已经通电并联网
- 您手里有摄像头的 RTSP 地址
- 设备与摄像头处于可互通的网络环境

如果页面右上角状态显示异常：

- `ZLM 流媒体` 不在线：通常表示视频流服务异常，无法正常预览摄像头画面
- `AI 推理服务` 不在线：通常表示智能识别服务异常，任务无法正常分析

建议先联系实施人员或管理员处理后再继续使用。

## 3. 推荐使用顺序

普通用户建议按下面顺序操作：

1. 在“设备管理”中添加摄像头
2. 在“任务管理”中创建任务并选择算法
3. 启动任务
4. 在“控制台”查看实时画面和告警概况
5. 在“事件告警”中查看、处理和删除告警
6. 如有需要，再配置语音报警、报警上传、定位管理等附加功能

## 4. 控制台

“控制台”用于快速查看整体运行情况。

您可以在这里完成这些事情：

- 选择 1 画面、4 画面、6 画面或 9 画面布局
- 在每个画面窗口中选择一个“正在运行的任务”进行预览
- 查看最近告警列表
- 查看今日告警数量和累计告警数量

说明：

- 控制台里只能预览“已启动”的任务
- 如果没有可选任务，通常表示还没有创建任务，或任务尚未启动

## 5. 设备管理

“设备管理”用于添加、修改、预览和删除摄像头。

### 5.1 添加摄像头

点击“添加摄像头”，填写：

- 设备名称：例如“前门摄像头”
- 安装地点：例如“1号仓库门口”
- RTSP 地址：例如 `rtsp://用户名:密码@IP:端口/...`

保存后，系统会自动尝试启动该摄像头的视频流。

### 5.2 查看摄像头状态

每个摄像头会显示推流状态：

- `推流中`：可正常预览
- `未启动`：流未启动
- `异常`：流启动失败或当前不可用

### 5.3 实时预览

点击摄像头后面的“实时预览”按钮，可以查看当前画面。

如果提示没有活跃流，请先点击“开流”。

### 5.4 常用操作

- `开流`：启动该摄像头的视频流
- `停流`：停止该摄像头的视频流
- `编辑配置`：修改名称、地点或 RTSP 地址
- `删除设备`：删除摄像头

注意：

- 修改 RTSP 地址后，系统会重新连接新地址
- 删除摄像头前，请先确认没有任务仍在使用它

## 6. 任务管理

“任务管理”是本系统最核心的页面。任务就是“让某个摄像头执行哪些智能识别规则”。

### 6.1 新建任务

点击“新建任务”，依次完成以下内容：

#### 基本信息

- 任务名称：例如“驾驶室疲劳监测”
- 摄像头：选择已经添加好的摄像头
- 备注说明：可选

#### 选择算法

系统会列出当前可用算法。勾选后即可为该任务启用。

常见算法示例：

- 离岗
- 闭眼
- 闭眼YOLO版
- 打哈欠
- 打哈欠YOLO版
- 未戴安全帽
- 未戴口罩
- 未穿救生衣
- 打电话
- 玩手机
- 吸烟

实际可见算法，以您当前系统页面显示为准。

#### 设置算法参数

不同算法会出现不同参数，常见有：

- 持续时间
- 跳帧数
- 置信度阈值
- 哈欠次数
- 冷却时间

这些参数决定告警灵敏度。一般情况下：

- 不确定时，可以先使用系统默认值
- 如果误报较多，可适当提高阈值或延长持续时间
- 如果漏报较多，可适当降低阈值或缩短持续时间

#### 设置报警条件

每个算法都可以设置报警条件：

- `全告警`：不区分状态，始终生效
- `航行中`：仅在航行中生效
- `停泊时`：仅在停泊时生效

如果您的现场不使用定位/航行功能，通常选择“全告警”。

#### 设置检测区域（ROI）

如有需要，可以点击“画检测区域”来指定只在某个区域内识别。

绘制方法：

- 左键单击：添加顶点
- 再次点击多个点：组成区域边界
- 点击第一个点：闭合区域
- 右键：撤销上一个点
- 完成后点击“确认区域”

如果不设置检测区域，则默认全屏检测。

适合使用检测区域的场景：

- 只检测工作台区域
- 只检测驾驶位或值班位
- 排除不相关背景区域

### 6.2 启动任务

任务创建完成后，在任务列表点击“启动”。

启动成功后：

- 任务状态会变为“运行中”
- 控制台可以选择该任务进行预览
- 系统开始自动分析并产生告警

### 6.3 停止任务

点击“停止”后，该任务将不再继续识别。

### 6.4 编辑任务

如果任务正在运行，系统会提示您先停止任务，再允许编辑。

这是正常设计，目的是避免运行中修改配置造成异常。

### 6.5 删除任务

删除前请确认该任务不再需要。删除后，该任务配置不会继续保留。

## 7. 事件告警

“事件告警”用于查看系统识别出的异常记录。

### 7.1 可以查看哪些内容

每条告警通常包含：

- 告警时间
- 任务名称
- 摄像头名称
- 算法名称
- 安装地点
- 报警截图
- 告警详情
- 处理状态

### 7.2 筛选方式

您可以按以下条件筛选：

- 任务
- 算法
- 状态
- 日期范围

### 7.3 两种查看模式

- 列表模式：适合批量查看和处理
- 卡片模式：适合看截图和快速浏览

### 7.4 处理告警

对未处理告警，可点击“标记为已处理”。

处理后状态会从“未处理”变为“已处理”。

### 7.5 删除告警

如果某条告警不再需要保存，可以删除。

也可以在勾选多条记录后进行批量删除。

说明：

- 删除后通常无法恢复
- 建议先确认截图和记录已不需要再删除

## 8. 语音报警

“语音报警”用于在告警发生时，通过外接设备或摄像头喇叭播放提示音。

### 8.1 开启语音报警

先打开“全局语音报警”开关。

### 8.2 配置发声设备

需要填写：

- 设备 IP
- 用户名
- 密码

这些信息通常由实施人员提供。

### 8.3 上传语音文件

系统当前只支持上传 `.pcm` 格式音频文件。

上传后可以在列表中看到：

- 文件名
- 文件大小
- 是否已被某个算法使用

### 8.4 为算法指定语音

在“算法语音映射”中，为每一种算法选择一个语音文件。

例如：

- 离岗 -> 播放“请勿离岗”
- 闭眼 -> 播放“请注意精神状态”

如果某个算法未配置语音文件，即使全局开关打开，该算法也不会播放语音。

## 9. 报警上传

“报警上传”用于把告警记录和截图自动发送到外部平台。

### 9.1 开启上传

打开“全局报警上传”开关后，系统会自动检查新告警并执行上传。

### 9.2 配置上传地址

需要填写：

- 上传地址：对方平台提供的接口地址

设备 ID 为系统自动生成，不需要手工填写，也不能手工修改。

### 9.3 上传特点

该功能支持：

- 自动上传新告警
- 失败后重试
- 网络恢复后继续上传

### 9.4 查看上传情况

在本页面可以看到：

- 待上传数量
- 已成功数量
- 上传失败数量
- 累计入队数量

还可以查看上传队列详情，包括：

- 报警 ID
- 识别类型
- 告警时间
- 任务
- 摄像头
- 当前状态
- 重试次数
- 错误原因

如果有失败记录，可以点击“立即重传失败项”。

## 10. 定位管理

“定位管理”用于查看设备当前位置，并根据速度判断当前处于“航行中”还是“停泊时”。

如果您的项目不涉及船舶、车辆或移动设备，可不使用此功能。

### 10.1 可查看的信息

- 定位状态是否有效
- 当前航行状态
- 定位来源
- 纬度、经度
- 对地速度
- 对地航向
- UTC 时间
- 北京时间
- 最近更新时间
- 错误信息

### 10.2 配置航行判定

可设置：

- 航行速度阈值
- 检测间隔

当速度高于阈值时，系统判定为“航行中”；否则判定为“停泊时”。

如果任务中的算法设置了“仅航行中报警”或“仅停泊时报警”，就会用到这里的状态。

## 11. 算法模型

“算法模型”页面主要给管理员或实施人员使用。

普通用户一般不需要频繁操作这里。

本页面主要分为三部分：

- 模型管理
- 算法配置
- 插件文件

### 11.1 普通用户需要知道什么

- 任务里能选到哪些算法，取决于这里是否已经配置好
- 如果任务页面没有您需要的算法，请联系管理员处理
- 模型、算法和插件属于系统级配置，错误修改可能导致任务无法运行

### 11.2 管理员常见操作

- 新增或编辑模型
- 上传模型文件和标签文件
- 新增或编辑算法
- 为算法关联模型
- 上传或删除插件文件

如果您不是管理员，建议不要自行修改。

## 12. 模型测试

“模型测试”用于临时测试某个模型在某个摄像头上的识别效果。

这通常用于调试，不是日常值守必需功能。

### 12.1 使用方法

1. 选择测试模型
2. 选择测试摄像头
3. 设置推理跳帧
4. 点击“开始测试”

开始后，页面会显示：

- 实时视频
- 检测框
- 目标数量
- 检测明细
- 推理耗时

### 12.2 适合什么时候用

- 想确认模型是否正常工作
- 想看检测框效果是否准确
- 想比较不同模型的识别表现

### 12.3 注意事项

- 页面离开后，本次测试会自动停止
- 浏览器异常关闭后，系统也会自动回收测试会话
- 该功能主要用于调试，不会替代正式任务

## 13. 系统管理

系统管理下目前包括：

- 系统信息
- 网络配置
- 日志查看

### 13.1 系统信息

这里可以查看设备运行状态，包括：

- 系统时间
- CPU 占用
- 内存占用
- 磁盘占用
- 磁盘剩余空间

适合用于判断设备是否运行压力过大。

### 13.2 网络配置

这里可以对 `eth0`、`eth1` 网口进行配置。

支持两种方式：

- DHCP：自动获取 IP
- 静态：手工填写 IP、子网掩码、网关、DNS

注意：

- 静态模式下至少要填写 IP 地址和子网掩码
- 保存网络配置后，设备网络可能立即变化
- 如果填写错误，可能导致网页断连
- 建议由熟悉网络设置的人员操作

### 13.3 日志查看

这里可以查看和重启相关服务，常见包括：

- 推理服务
- Python 算法服务
- Go 后端服务
- ZLMediaKit 服务
- FRPC 服务

适合在故障排查时使用。

如果您只负责日常值守，一般不需要操作这里。

## 14. 常见问题

### 14.1 为什么摄像头添加后看不到画面

可能原因：

- RTSP 地址错误
- 摄像头网络不通
- 用户名或密码错误
- 摄像头未开流
- 流媒体服务异常

建议先检查：

- 设备管理中的状态是否为“推流中”
- 页面右上角 `ZLM 流媒体` 是否在线

### 14.2 为什么任务启动失败

常见原因：

- 摄像头流不可用
- AI 推理服务未运行
- 算法或模型配置不完整

建议先检查：

- 摄像头是否能正常预览
- 页面右上角 `AI 推理服务` 是否在线
- 算法模型是否已由管理员配置好

### 14.3 为什么没有产生告警

可能原因：

- 任务没有启动
- 检测区域画错了
- 报警条件设置成了“航行中”或“停泊时”，当前状态不满足
- 算法参数过严，阈值太高或持续时间太长

建议排查：

- 控制台是否能看到任务画面
- 任务状态是否为“运行中”
- ROI 是否限定得过小
- 先尝试使用默认参数

### 14.4 为什么语音报警没有声音

可能原因：

- 未开启全局语音报警
- 没有配置发声设备
- 算法没有绑定语音文件
- 上传的音频文件不是 `.pcm`

### 14.5 为什么报警上传失败

可能原因：

- 上传地址错误
- 对方平台未开启
- 网络不通
- 对方接口格式或权限不匹配

可以到“报警上传”页面查看失败原因，并在问题解决后执行重传。

## 15. 日常操作建议

推荐的日常使用方式如下：

- 上班前：打开控制台，确认服务在线、任务正常运行、画面正常
- 巡检时：到事件告警页面查看未处理告警并及时处理
- 调整现场时：如摄像头位置变化，及时检查任务 ROI 是否需要重画
- 网络或设备变更后：检查摄像头是否仍能正常预览，任务是否还能正常启动

## 16. 建议的权限分工

为了减少误操作，建议按角色分工：

- 普通值守人员：控制台、事件告警
- 业务操作人员：设备管理、任务管理、语音报警、报警上传
- 管理员/实施人员：算法模型、模型测试、网络配置、日志查看

## 17. 一句话总结

如果您只想尽快学会使用本软件，请记住这条主线：

先添加摄像头，再创建任务并启动，然后到控制台看画面、到事件告警看结果；语音报警、报警上传、定位管理等功能按需要再开启。
